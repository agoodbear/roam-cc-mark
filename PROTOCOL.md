# CC 讀取／回寫協定（請CC修改）v2

> 定位：這是 Bear 的「紅筆批註」工具。**紙是 Bear 的，紅圈是 Bear 畫的；CC 只能在圈裡寫鉛筆字（提案），Bear 點頭才變墨水。**
> CC（Claude Code，透過 Roam MCP 從外部）改稿前**先讀完這份**。原稿範圍 = **一個 Roam page，一頁一篇**。

## 一、憲法（凌駕一切流程，違反即中止）

1. **CC 不得直接 `update_block` 任何原稿 block。**（原稿＝Bear 的字）
2. **CC 不得刪除 Bear 建的標記 block。**（清除只由 Bear 的 ✅ 觸發，透過 extension）
3. **CC 不得在原稿區新增 block。**
4. CC 的所有產出，只能寫進「**標記 block 本身（提案欄）**」——換句話說，CC 的字永遠關在標記裡、帶著身分，不匿名混入正文。
5. Bear 是唯一有權讓字進入原稿的人。

## 二、標記格式

Bear 建立（待處理）：
```
#請cc修改 【指令】<意圖>：<一句話> 【第N處】 【原文】「<精確片段>」
```
- 意圖 ∈ {潤, 接, 查, 議}（見四）。`【第N處】` 只在原文重複時出現，指第幾個出現。無【原文】= 整段。

CC 回寫（待審）＝把 tag 從 `#請cc修改` 換成 `#cc提案`，並補欄位，**欄位順序固定**：
```
#cc提案 【指令】…【第N處】【原文】「…」【提案】「<替換/草稿文>」【備註】<一句說明/來源>
```

## 三、一次讀全（單一 MCP call，別逐個撈）

待處理（Bear 剛丟給你的）：
```clojure
[:find ?childUid ?markStr ?parentUid ?parentStr
 :where
   [?t :node/title "請cc修改"]
   [?c :block/refs ?t] [?c :block/uid ?childUid] [?c :block/string ?markStr]
   [?p :block/children ?c] [?p :block/uid ?parentUid] [?p :block/string ?parentStr]
   [?p :block/page ?pg] [?pg :node/title "<頁名>"]]
```
把 `"請cc修改"` 換成 `"cc提案"` 即撈「已提案待審」的。Bear 也可在 extension 右下點 📋 打包成一段文字（含 page uid）貼給你，你就零查詢開工。

## 四、協作意圖（四種，各有明確回寫形態）

| 意圖 | Bear 要的 | CC 做什麼 | 回寫 | 守住 Bear 的聲音 |
|---|---|---|---|---|
| **潤** | 這句照我意思改好（口語化/縮短/去AI腔/台灣用語都算） | **最小編輯距離**改被圈的字 | `【提案】「替換文」`；tag→cc提案 | 只動被圈的字，保留 Bear 用詞、句式、判斷語氣；**禁擴寫禁加料、禁加 AI 過場語**；改幅超出指令寧可退回問 |
| **接** | 我卡住了，幫我起一段草稿 | 讀整頁上下文 + 風格檔 `reference_ecg_writing_style_profile` 寫草稿 | `【提案】「草稿」`；tag→cc提案 | 這是素材不是成品；被接受後會自動掛 `#cc草稿`＝還不是 Bear 的字，等 Bear 改寫收編 |
| **查** | 這數字/引用/說法可靠嗎、補來源 | 查證（**醫學數字照鐵律：兩獨立路徑交叉、走 OpenEvidence**），**一字不改原文** | `【備註】結果＋來源`；tag→cc提案 | 完全不碰原文；怎麼改字是 Bear 看完自己動手；查不實寫「待確認」不編造 |
| **議** | 給我 2–3 個說法選項／順序建議 | 給選項或結構建議，不動原文 | `【備註】編號清單`；tag→cc提案 | 只出主意；採納與改寫都 Bear 來 |

三種回寫形態：**替換型**（潤）、**插入型**（接）、**註記型**（查/議）。看到意圖就知道你的產出該長什麼樣、界線在哪。

## 五、對帳清單（回寫完，在 chat 回一份，Bear 一眼知道回 Roam 要看什麼）

一處一行：
```
#3 潤｜「到院前已去顫三次」→ 建議「到院前電擊了三次」（12字→10字）
#5 查｜D2B 90 分鐘 → 已證實，來源：2013 ACCF/AHA STEMI Guideline
#7 接｜（第 4 段結尾）已補一段 82 字草稿，接受後請改寫收編
```

## 六、生命週期與收斂

```
#請cc修改（Bear 建，待處理）
   → CC 回寫 → #cc提案（待審，含【提案】/【備註】）
      → Bear ✅ 接受 → extension 套用（潤=替換原文第N處／接=插入新 block 掛 #cc草稿／查議=只刪標記）
      → Bear ↩ 退回 → 改指令 → tag 換回 #請cc修改 → 再送（面板會顯示你上一版提案當參考）
      → Bear ✎ 改進這句（查/議專用）→ 轉成潤稿請求，指令已帶上你查到的來源/建議
         （例：潤：把查證到的來源整合進這句：<來源>）→ 再送 → CC 提「原文＋來源」整合好的句子
（標記消失＝這處收斂）  全頁標記＝0 ＝ 定稿
```

## 七、定稿轉 Hugo（兩個歸零條件）

Bear 說「〈頁名〉轉 Hugo」時，CC 先檢查：**① 標記＝0 ② `#cc草稿`＝0**（未收編的 CC 草稿）。
- 不滿足 → 列出擋下，不轉。
- 滿足 → 讀整頁產 `content/posts/<type>-post-N.md`（沿用 ecg/study/travel/erlife-post-N 慣例；**Hugo 禁 H1**；排除「🗂 素材」子樹；跑 zhtw-mcp lint）→ 部署 → 回 Roam 頁首寫一行 `✅ 已發佈 → <url> <日期>`，該頁視為封存（之後小修直接改 Hugo，大改才重開 Roam 頁）。
- 若有明示保留的 `#cc草稿` 段落 → 轉稿前必過 `/de-ai-zhtw`（只掃 CC 草稿出身的段落，Bear 原文絕不進）。

## 八、聲音守則（獨立成節，最高優先）

- **潤**：最小編輯距離、保留 Bear 用詞與判斷句、禁 AI 過場語與擴寫。
- **接**：先讀 `reference_ecg_writing_style_profile`；產出自知是草稿，不假裝成 Bear 的定稿。
- **查**：醫學數字兩路徑交叉、走 OpenEvidence；查不實寫「待確認」，禁編造引用/數字。
- 一切以「這是 Bear 的文章、Bear 的聲音」為最高判準。CC 是紅筆，不是作者。
